First, Special thanks to Jim Storer for permission to recreate this classic simulation from the early days of computing.

In 1969 the Apollo moon landing was dominating the news and Jim was a young high school student with rare access to a computer and the result was Lunar Lander.


The following information will allow you to learn more about LUNAR LANDER as well as the hardware and software used to bring it to life in 1969.  Given the state of computer gaming in 1969 you can see how much things changed when the 1973 version of Moonlander was released.

Q. What was the hardware used to play Lunar Lander?

A. Lunar Lander was developed on a Digital Equipment Corporation (DEC) PDP-8 with a Teletype Model 33 as the console terminal (it would be a few more years before video display terminals became widely available). The PDP‑8 was a time-sharing minicomputer with a 12-bit architecture and each terminal had 8K of memory to hold the Focal interpreter and the Lunar Lander source code.

We have tried to create the speed and sound of the original environment but you can see that extended game play took up a lot of time and it is not easy to land successfully.

Q. How big was the original Lunar Lander application?

A. The Lunar Lander Focal source code is quite small, still it had to be trimmed of all white space and comments to get it to load on the PDP‑8 simulator.  Memory was limited on the PDP-8 so Jim wrote terse code to keep the size of the code small. Single letter variables and using Focal keyword abbreviations like 'L' for 'LET' and 'T' for 'TYPE' were used to limit the size of the code.

Q. What were some of the challenges in recreating Lunar Lander?

A. Learning how to program Focal was not difficult since it was related to the popular BASIC programming language and the code is very small, even if it is quite terse.  The only problem encountered during the recreation effort was a numerical instability that made the output differ from the original listing provided by Jim.  Much more time was spent to recreate the simulation of a Teletype console that would accurately reflect the look of the original.

Again this port would not have been possible without the work of the simh project and the PDP-8 simulator and the work of computer historians like bitsavers.org to make the Focal language documents available on the web.
